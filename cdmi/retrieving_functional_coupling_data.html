The Issue of Retrieving Functional Coupling Data

As we proceed to improve annotations of gene function, precision of metabolic models, and estimates of regulatory architecture, we will need access to the clues that support estimates of "functional coupling". In this short example, we allow the user to designate a genome within the set stored in the CS (presumably a prokaryotic genome), and we retrieve data relating to co-occurrence (in close proximity on multiple genomes) and co-expression (when we have data available ). Here is the short program that accesses both the co-occurrence data and the co-expression data: 
use CDMI_APIClient;
my $kbO = CDMI_APIClient->new;

my $g = shift @ARGV;  # Just grab the first and only argument - it should be a Kbase genome

my $genH       = $kbO->genomes_to_fids([$g],['peg']);
my $fids       = $genH->{$g};
my $funcH      = $kbO->fids_to_functions($fids);
my $fid_occH   = $kbO->fids_to_co_occurring_fids($fids);
my $fid_expH   = $kbO->fids_to_coexpressed_fids($fids);

foreach my $fid1 (@$fids)
{
    my $co_occur = $fid_occH->{$fid1};
    my $co_expr  = $fid_expH->{$fid1};

    my %related;
    if ($co_occur && (@$co_occur > 0))
    {
        foreach my $tuple (@$co_occur)
        {
            my($fid2,$sc) = @$tuple;
            $related{$fid2}->[0] = $sc;
        }
    }

    if ($co_expr && (@$co_expr > 0))
    {
        foreach my $tuple (@$co_expr)
        {
            my($fid2,$sc) = @$tuple;
            $related{$fid2}->[1] = $sc;
        }
    }

    my @related_fids = keys(%related);
    foreach my $fid2 (keys(%related))
    {
        if ($fid2 gt $fid1)    # avoid duplicating based on order
        {
            $related{$fid2}->[1] = $sc;
        }

        my($occ,$pcc) = @{$related{$fid2}};
        if (defined($occ) && ($occ >= 30) && defined($pcc) && ($pcc >= 0.7))
        {
            $pcc = sprintf("%0.3f",$pcc);
            print join("\t",($fid1,$fid2,$occ,$pcc,$funcH->{$fid1},$funcH->{$fid2})),"\n";
        }
    }
}
The program begins by accessing the protein-encoding genes, their functions, and the functional coupling data. 
my $genH       = $kbO->genomes_to_fids([$g],['peg']);
my $fids       = $genH->{$g};
my $funcH      = $kbO->fids_to_functions($fids);
my $fid_occH   = $kbO->fids_to_co_occurring_fids($fids);
my $fid_expH   = $kbO->fids_to_coexpressed_fids($fids);
It then goes through the genes gathering the related genes for each of the values of $fid1.

The first two columns are the possiblyu related PEG IDs. The third column is a count of the number of distinct OTUs in which pairs of genes similar to these two occur close to one another. The fourth column gives the Pearson Correlation Coefficient. The fifth and sixth columns contain the functions asserted for the two genes. Note that the lines we are showing have very strong correlation values.

We might ask "What is the strongest evidence relating to a gene of unknown function?" Here is certainly a good candidate: 
kb|g.0.peg.3658 kb|g.0.peg.4040 580     0.861   DNA polymerase III subunits gamma and tau (EC 2.7.7.7)	FIG000557: hypothetical protein co-occurring with RecR
Note that kb|g.0.peg.4040, whic is annotated as 
     FIG000557: hypothetical protein co-occurring with RecR
occurs close ot a gene annotated as 
     DNA polymerase III subunits gamma and tau (EC 2.7.7.7)
and it occurs close in genomes occurring in 580 distinct OTUs. The Pearson correlation coefficient based on expression profiles is 0.861. You should ne able to use this simple program, run it on genomes in the KBase for which expression data exists, and then extract lines containing the word "hypothetical". If you sort them by the third and fourth columns, you will see some astonishingly suggestive clues.

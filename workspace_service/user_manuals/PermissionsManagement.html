Managing workspace permissions

Now that you can create your own workspaces, you will want to manage the permissions for the access of those workspaces by other users. Any user with administrative privileges can manage workspace permissions. The workspace owner has administrative permissions by default, but other users can be provided with administrative privileges if required. There are three commands to support the management of workspace permissions:

  1. kbws-setglobalperm: resets the global default permissions for all users for a workspace
  2. kbws-setuserperm: resets specific permissions for a list of users
  3. kbws-perm: returns a list of all currently set user-specific permissions for a workspace

1. kbws-setglobalperm


	kbws-setglobalperm  [-?ehw] [long options...]
		-w --workspace     ID for workspace
		-e --showerror     Set as 1 to show any errors in execution
		-? -h --help       Print this usage information



	$ kbws-setglobalperm w
		Global permissions reset!

2. kbws-setuserperm


	kbws-setuserperm   [-?ehw] [long options...]
		-w --workspace     ID for workspace
		-e --showerror     Set as 1 to show any errors in execution
		-? -h --help       Print this usage information



	$ kbws-setuserperm kbasetest r
		User permissions set for:
		kbasetest

3. kbws-perm


	kbws-perm [-?ehw] [long options...]
		-w --workspace     ID for workspace
		-e --showerror     Set as 1 to show any errors in execution
		-? -h --help       Print this usage information

Let's see an example of this command in action. ....


	$ kbws-perm
		Username  Permissions
		kbasetest r          
		public    a          
		default   w

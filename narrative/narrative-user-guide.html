Narrative Interface Quick Start Guide

Before Beginning

  • Sign up for a KBase account, if you haven’t already.
  • We recommend accessing the Narrative Interface and other KBase interfaces with the latest version of either Mozilla Firefox or Google Chrome browsers. Please see the Supported Browsers page for more information.

The Narrative Interface is a new graphical user interface that enables researchers to create interactive, reproducible records of the data, computational steps, and thought processes underpinning their results. These records can be shared as “active papers” called Narratives that let others repeat the computational experiment, even altering parameters or input data to achieve different or improved results. To learn more about the features of this interface, visit the Narrative User Interface Overview.

Although the Narrative Interface is still in early development, new capabilities and features are constantly being added, and users can begin to explore what is currently available using the prototype version available at narrative.kbase.us. This guide will help you get started by showing you how to:

  • Log in to the Narrative Interface
  • View a Featured Narrative
  • Copy a Narrative
  • Create a new Narrative
  • Add content to your Narrative

Log in to the Narrative Interface

Go to the Narrative Interface home page at narrative.kbase.us. At the top, you will see these key features:

  • KBase logo — Links to the KBase home page (kbase.us).
  • Narratives — From the “Narratives” menu, you can create a new Narrative and access your existing Narratives, as well as others that have been shared with you by collaborators or made publicly available.
  • Workspaces — In KBase, workspaces let you store, analyze, and share your private data objects. The “Workspaces” menu links to Workspace Browser, where you can create and manage workspaces and view interactive information about the data objects stored in them.
  • Search Data — The new search interface, which is rapidly developing, allows you to search some of the public data stored in KBase. More data types will be added to the search interface soon.
  • Upload — Once developed, this tool will let you upload data files from your computer to your workspace so that you can analyze them in your Narratives.
  • Help — Links to documentation and contact information for submitting questions and help requests.

Using either the link at the top right or the box in the center of the page, sign into KBase with your user account and password.

View a Featured Narrative

Once signed in, you will be taken to the Featured Narratives page (see image). These Featured Narratives demonstrate some of the types of systems biology questions that can be explored in KBase.

Error 502

Sometimes, instead of seeing a Narrative, you will see an error message that says "502 Bad Gateway Error". The Narrative team is working on resolving this problem. In the meantime, if you reload the page, you should be able to see the Narrative you selected.

 

Click on one of the Featured Narratives to open a read-only version of the Narrative that has already been populated with a summary, input datasets, and output from KBase services that have been run on the input data. The screenshot below shows part of the Comparative Genomics Narrative.

When you open a Narrative, there are several key features:

  1. The "Data" panel in the upper left for viewing all data files in your Narrative
  2. The "Services" panel in the lower left for selecting different KBase services that you can use to analyze your data
  3. The main center panel for viewing or composing a Narrative
  4. A URL that is unique for every Narrative, allowing you to easily share your Narrative with collaborators or even the scientific community at large

Why is the Narrative complaining about websockets?

The KBase Narrative Interface is based on the iPython Notebook, which uses a protocol called "websockets" for communicating between the browser and the server. The websocket protocol is similar to, but not the same as, the HTTP protocol used for the Web. Thus, some firewalls and Virtual Private Networks (VPNs) that work fine for web browsing can still block or interfere with websockets, causing the Narrative to fail.

If you are trying to use the Narrative Interface and see error messages about websockets, you have two options: (a) stop using the VPN or firewall that is causing it to fail or (b) change the VPN or firewall to properly handle the websocket protocol. Option (b) will probably involve talking to your local network administrators.

One other thing to check is that the URL you are visiting starts with https://. If it starts with http://, try changing it to https://.

Another reason you might see a websocket error is in a Narrative window that you've left open for a while and your session has expired. If reloading the page doesn't work, you may have to log out and log in again.

If you are not sure whether the errors you are seeing are due to websockets, you can test the websocket protocol on your connection to the Internet using the public server at http://www.websocket.org/echo.html.

The Data Panel

Click on the “Narrative” tab in the data panel on the upper left side of the page. This tab shows the datasets that are used in the current Narrative. Note that this panel (see image) can take a while to load when you first open the Narrative.

Several options are available for finding and displaying data objects in the data panel. A drop-down menu will let you sort data by type. By default, all types are shown when you open your Narrative.

You can also list data objects by ID in alphabetical or reverse alphabetical order by clicking the ID button in the middle of the panel.

A search box lets you search for a data object of interest within the data panel. For example, type “model” in the search box to display a list of models that were generated in this Featured Narrative. Your search will return a list of five data objects:

  • CACIA_14H1_model_translated_gf.fba.0
  • CACIA_14H1_model_translated.fba.0
  • CACIA_14H1_model_translated
  • CACIA_14H1_model_translated_gf
  • CACIA_model_kbase

As for all data objects in the panel, you can hover over the names of these models to see a short description of the type of object they are. For more information, click on the question mark to the right of a data object. This will open a window listing the data object’s properties. Below is the properties window for CACIA_model_kbase.

 

Among the information in this window is a drop-down menu at the bottom that shows the object’s version history. Also at the bottom are two buttons that link to further details.

The “Object Type Details” button opens a window that displays information about what type of data you’re viewing and about data versioning.

The “View Landing Page” button opens a new browser window showing a Landing Page with more information about the object. The information shown will depend on the type of object you have selected. Although not all data objects in KBase have Landing Pages yet, the number that do is increasing. The panels on Landing Pages are moveable so you can customize the layout of the page. In this example, the selected object is a flux balance analysis (fba) model, so the Landing Page provides overview information on the model, as well as other model details such as reactions and compounds. A graphical representation of the metabolic pathway is also provided. (Note that information on some Landing Pages may take a few minutes to load.) More details on Landing Pages are provided in the Data Search Quick Start.

Return to the browser window displaying the Featured Narrative and close the data object properties window. As you look through the Narrative, you can follow the analyses that were run by the creator of the Narrative, as well as read commentary about why these analyses were run and what the results imply.

Copy a Narrative

If you want to try running the steps in a Narrative and possibly even changing them, you will first need to make a copy of it. You can copy any Narrative for which you have read permission. Once you copy a Narrative and its associated data into your own workspace, you can run all of its analyses, alter parameters or datasets if you choose, and save your changes to your copy.

To copy a Narrative, start from the page where you are viewing the Narrative that you are interested in. Click the “Workspaces” link at the top of the page to access the Workspace Browser. The workspaces that you have read or write access to are listed on the left side of the page. The one that holds the Narrative that you were looking at will appear at the top of the list with a blue background.

Hover your cursor over the workspace name that has the blue background; a gear icon will appear (see image) that will allow you to clone the workspace, including all of the Narratives and data it contains.

Clicking the gear will bring up a “Manage Workspace” window. At the bottom of this window is a “Clone” link (see below).

Once you click this link, you will be prompted to choose a name for your new cloned workspace. Note that a workspace name can contain letters, numbers, or underscore characters, but empty spaces and other non-alphanumeric characters are not allowed. In this case, we chose “Featured_narratives_copy.” By default, the global permission for your new workspace is set to “none,” but you can change it to “read” to allow others to view your workspace. After choosing a name for your workspace, click the blue “Clone” button at the bottom of the window.

You now will see your cloned workspace at the top of the workspaces list on the left (see image below). When you cloned the “MicrobesFeaturedNarratives” workspace, that made a copy of all the Narratives and data objects that it contained. This workspace contained three other Narratives in addition to the Comparative Genomics Narrative you viewed earlier in this tutorial. You can see these other Narratives and browse through the data objects associated with each one by using the page navigation options at the bottom of your list of data objects.

Narratives are distinguished from other data types in the list by their bold and italicized names. The “Type” column also tells you at a glance which data type an object is. If there are many objects in a workspace, you may want to use the pulldown list of types (which defaults to "All Types") to show only the Narratives.

Another way to locate your copy of the Narrative is to click on the "Narratives" link at the top of the page and then the "My Narratives" link on the left side. Your new cloned Narrative should appear at the top of the list (which is sorted by date, with the most recently created or copied at the top).

Now that you have a copy of the Comparative Genomics Narrative, you can try running the steps in it or continue to the next section to learn how to recreate its steps in your own new Narrative, which will allow you to work through the Featured Narrative’s analyses and familiarize yourself with many aspects of the Narrative Interface.

Create a New Narrative

To create your own new Narrative, click the “Narratives” link in the top menu to access the Narratives page. At the top of this page, find the “+New Narrative” button.

Clicking on this button will open the “Create New Narrative” window (see image). Choose the workspace you want to put your new Narrative in by selecting one of your existing workspaces from the drop-down menu. You can also click the blue “New WS” link to create a new workspace for your Narrative. Here, we chose the cloned workspace that we created in the previous step of this tutorial because it contains all the data objects in the Featured Narrative we’re interested in recreating.

Once you have designated a workspace for your Narrative, enter a name for it. Here, we chose “My_new_narrative.” Add an optional description if you wish and then click the blue “Create” button.

You will see your new Narrative listed at the top of the My Narratives page (see image).

This page has several options for displaying and managing your Narratives:

  1. This drop-down lets you customize the columns of information displayed in your list of Narratives. By default, Narrative, Workspace, Last Modified, Modified by, and Shared With are displayed. Additional options include Timestamp, Size, Byte Size, and Module.
  2. The trash bin icon, along with the two buttons to its right, appears when you select a Narrative from the list. Clicking the trash bin will delete your selected Narrative. An undelete option is coming soon, so be aware that Narrative deletions are currently permanent.
  3. The open folder icon lets you copy a selected Narrative to a workspace you own or have admin privileges for.
  4. This button allows you to rename a selected Narrative.
  5. A search box lets you search for a specific Narrative, a useful option for quickly finding Narratives in a long list.
  6. Each Narrative is followed by a version number. Click on the number to see the history of the Narrative.
  7. Select or hover over a Narrative name to display a blue “More” link to the right. Clicking “More” will open a window that displays information about the Narrative data object including size, type, ID, owner, modified date, and more.
  8. As in Workspace Browser, the gear icon here will open the “Manage Workspace” window. In addition to cloning a workspace, this window provides options for sharing. Click the "Edit" button in the upper right corner if you want to assign specific read, write, or admin privileges for individual users. Currently, you need to enter each KBase user name if you want to set individual user permissions. In the future, there will be a search feature for finding other KBase users. The "Manage Workspace" window also has a button for deleting a workspace. Keep in mind that you cannot undo deletion of a workspace.

Now that you’ve explored some of the features of the My Narratives page, find your new Narrative in the list and click on it to open your empty Narrative:

Notice that the data panel in your new Narrative is already populated with data objects that were copied from the Comparative Genomics Featured Narrative. When you create a new Narrative in a new workspace, the data panel will be empty until you add data using KBase’s search interface or upload your own data. In the alpha version of the Narrative Interface, the tools for uploading your own data are currently under development.

Save Your Work!

We are working on enabling autosave for Narratives, but for now, don't forget to click the Save button (the leftmost button) to save your Narrative.

Add Content to your Narrative

Narratives are composed of an ordered series of sections, or “cells,” that store your analysis steps and commentary. Cells can contain text (to help you record hypotheses, comments, or ideas), or they can hold an analysis step. “Input" cells (with blue borders) let you invoke one of KBase’s analysis functions, with input data and parameter settings of your choosing. (Empty input cells say "In [ ]" to the left--"In" stands for "input", and the empty brackets will later contain a number representing the number of times that cell has been run.) "Output" cells (with gray borders) show the output generated from an input cell.

At the top of the Narrative page is a toolbar with a row of buttons that you can use to add cells to or edit your Narrative. If you use your mouse to hover over a button, a short description of the button's function will appear. (Please note that in this alpha version, the buttons may not all work as expected.) Descriptions of these buttons, along with their keyboard shortcuts, are provided below. The keyboard shortcuts will allow you to add content to and edit your Narrative without having to scroll to the toolbar at the top. You can access a list of all keyboard shortcuts at any time in the Narrative Interface by using "Ctrl-m h" (press and hold the "Control" and "m" keys, release, and then press "h").

From the drop-down menu next to the row of buttons, you can select the type of cell to add to your Narrative. The Markdown cell type stores formatted text and can be used to add annotations or descriptions within the Narrative. You can also choose to add cell types for entering raw text or adding headings to your Narrative. In the screenshot below, we selected "Markdown" from the drop-down menu, typed some text into the cell, and then double-clicked outside the cell.

To exit a cell, simply click outside the cell or press the Shift+Return keys to take you out of the selected cell and into the next cell.

Recreating the Steps in a Featured Narrative

Opening the Featured Narrative in a new browser tab will be useful as you recreate the analysis steps in your own Narrative. To do this, click on “Workspaces” in the top menu and find the “Comparative_Genomics” Narrative in the list. Click on it to open it in a new tab. The screenshot below shows the first step in this Narrative and the first input cell: Build Genome Set Object.

Return to your new Narrative by clicking the tab that will take you back to Workspace Browser and your list of Narratives. Find your new Narrative in the list and open it.

To add a new analysis to your Narrative, you can select a function from the “Services” menu on the lower left. Each function has a “?” button next to it that you can click for information about that function.

If you click on a function name, a new input cell will be added to your Narrative, with fields for you to fill in to set the parameters for that analysis.

To add the “Build Genome Set Object” input cell to your Narrative, click on "Microbes Comparative Genomics" in the "Services" panel and scroll down unto you find it. Click on it to add it to your Narrative.

In the blue top border of the cell, a description of this service was displayed by clicking the "..." button to the right of the function name.

To the left of each text box in an input cell you will see the name of that field (for example, "Output genome set ID"). The text to the right of the box provides more information about what you can enter in that field. When you click on the field for a given input cell, KBase recognizes the data objects from your workspace that are appropriate inputs for that cell and presents you with a drop-down list of options to choose from. In this case, select the first option, Rhodobacter_external_genomes.

When you have entered the parameters for the analysis, click the Run button in the lower right of the cell to execute the function with your supplied parameters. If the run succeeded, you will see an output cell with a gray border. In the top right corner of the output cell border is the time and date when the run completed.

You can continue recreating the analysis steps from the Featured Narrative, altering parameters and data if you choose.

The next section will provide instructions on using KBase’s search function to load data to your workspace and then analyze it in the Narrative Interface.

Searching for Data and Analyzing it in a Narrative

You can load data selected from your search results to any workspace you own or have admin privileges for. For this step, we will add data to our “home” workspace, which KBase automatically generates for you when you sign up for an account.

To access KBase’s search interface, click on “Search Data” at the top of the page.

The Data Search Quick Start has detailed instructions on how to search, sort, filter, and transfer data objects to your workspace for subsequent analysis with KBase tools. Please refer to it for in-depth information.

Here, we will do a simple keyword search by typing “Escherichia coli ” in the search box and pressing Enter or the search icon. Our search returns results in three categories of currently searchable data.

Click on “Genomes” to access the list of search results for this category.

From this list, use the checkboxes in the first column to select the first genome, Escherichia coli TW10828. Notice that when you selected this genome, it was added to your Data Cart on the left. The number above the cart indicates that you have one data object in your cart. The number will increase as you select more data objects to add.

Once you are finished searching and selecting data, you are ready to transfer your data objects to a workspace of your choosing.

To transfer your selections, click the blue button below the Data Cart to bring up the cart view:

You must first choose the workspace you want to transfer your data objects to. For this example, choose your “home” workspace.

Once your workspace is selected, click on the blue “Genomes” label in the middle of the page to see the data objects you selected from your search results. Checkboxes will appear beside the data objects. Use these boxes to choose the data you want to transfer to your workspace. Once selected, click the blue “Copy” button that appears below your workspace name.

Close the Data Cart view by clicking the “x” in the right corner.

Return to Workspace Browser by clicking “Workspaces” in the top menu and locate your home workspace to verify that your E. coli genome (with a KBase ID of kb|g.3696) was transferred to your workspace.

You can now use data from your search results in analysis steps in the Narrative Interface.

Start by creating a new Narrative in your home workspace or open an existing one.

Notice the kb|g.3696 genome is listed in the data panel of your Narrative. In this example, we are going to view of list of genes in this genome by using the “View Genome” widget under “Microbes Assembly & Genome Import” in the Services panel (see image). Find this widget in the panel, and click on it to add the "View Genome" input cell to your Narrative.

In the input cell, select the kb|g.3696 genome from the drop-down list and click Run. In the resulting output cell, select the Genes tab to see the list of genes for this E. coli genome.

 

The Narrative User Interface and this documentation are both evolving rapidly. Please check back soon for more information.

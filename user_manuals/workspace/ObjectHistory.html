Viewing object history and reverting objects

Now that you can create and manage the objects in your workspace, you will want to learn about the workspace features that support object provenance. When an object in a workspace is overwritten, the old version of the is not lost. It is conserved, and a history of previous versions of all workspace objects is maintained. At any time, you can retrieve any previous version of an object, and you can revert the current instance of an object to any of its previous versions. There are three commands in the workspace that provide support for object history:

  1. kbws-history: retrieves the metadata for all previous versions of an object
  2. kbws-revert: reverts the current instance of an object to a previous version
  3. kbws-get: enables the user to retrieve the current and all previous versions of an object

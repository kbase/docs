Walk-thru of the KBase Notebook



1. Login with your KBase credentials in order to create, view and save your private notebooks.


2. Create a new notebook if you like.


3. To get started you might want open an existing KBase notebook.


4. You can also create a copy of an existing notebook.


5. This notebook shows you how to examine sequence quality information in KBase. Reading thru this notebook can give you an idea of how analyses can be performed and displayed in KBase.


6. In the top right-hand corner you can click on the "Analysis Builder" button to open the visual analysis builder. This is a convenient way for any user to get started, but especially those without prior programming experience.


7. Before we go any further, try creating a new notebook by clicking the "+" symbol next to the tab of the notebook that you already opened.


8. Then, give your new notebook a name and click "Launch".


9. The "import data" tab in the Analysis Builder is where you import the metagenomes that you're interested in analyzing. Below you can see we are viewing all available metagenomes by their "sequencing type" but have restricted our search set to include only WGS (Whole Genome Shotgun) metagenomes. By selecting the search field from the drop-down menu and then typing in your search criteria and hitting enter, you can enter multiple search criteria for different search fields.


10. Notice in the following picture, that while we are still viewing only WGS metagenomes, we have changed the field to "id" and entered the beginning of our metagenome sample ID to narrow our list of metagenomes. Then, we selected the two metagenomes we are interested in (mgm4440275.3 and mgm4440279.3) and clicked on the right arrow button to add them to our selected metagenomes list.


11. Next, clicking on the green checkmark loads the data for these metagenomes. You will see that the python code for loading this data is automatically generated in the notebook below. The line numbers to the left of each cell in the notebook keep track of the order in which the cells were run. Sometimes loading the initial data can take a few minutes. During this time, instead of a line number you will see [*] that will indicate the command is still running. Once the code in that line has finished running, the line number will appear in place of the asterisk.


12. Now we will click on the "select data" tab and select what we are interested in analyzing. Perhaps we only want to look at a subset of the samples we loaded. We also need to select the type of data we are going to visually analyze. Once we are ready, we again click on the green checkmark to generate the python code necessary for performing this subselection. Note that our subselection will be loaded into the variable listed under "select target -> variable name".


13. Finally we want to go to the "visualize data" tab to generate our visual analysis. There are various analysis types to choose from at the top of this tab. Below, we have selected a heatmap with the default settings. Note that you must also select which variable from the "select data" step that you would like to analyze under "available variables". In our case there is only one to choose from, but you can have multiple. When we're ready, we again click the green checkmark.


14. The next slide shows the graph that was generated inline which we can view by scrolling down in our notebook.


15. Scrolling to the top of the page, you will find a green button (with an eye on it) next to the "Analysis Builder" button, which will open a new browser window containing only the visual analyses generated in the notebook.


16. This last slide shows what this new window would look like in our example.

NAME

Bio::KBase::FunctionalOrthologPredictor::Client

DESCRIPTION

This module provides public interface/APIs for KBase functional ortholog predictor. The function is first demonstrated by Mutwil et al., 2011, Plant Cell. Given a well studied gene in Arabidopsis and a list of potential ortholog in Poplar, this module can predict the best functional ortholog based on gene co-expression network. Before using this module, make sure you can use KBase networks API and plant_expression service.

Here is how it is working: Given a well studied gene_A in a model plant(Arabidopsis), we have a list of candidate orthologs in another less-well studied plant(Poplar). This list may be identified by BLAST, or other sequence-based similarity search. Now, we want to know which one of the potential ortholog has the best chance to perform the same biological function as gene_A does.'rank_pgenes', means rank the list of potential genes. First, get the co-expression neighbors of gene_A and all the potential genes through KBase networks API. (Currently which co-expression network dataset is used in hard coded. Later, this will be selected by the user). Then, the PFAM domain information for all these genes is retrieved.(Since there is not a KBase-wide PFAM domain table, the PFAM domain table for Arabidopsis and Poplar are stored in devdb1.newyork.kbase.us temporarily.). Finally, the shared PFAM domain between gene_A and each of the potential gene is compared. A score is calculated based on: number of shared domain bewteen ((the neighbors of gene_A and the neighbors of potential gene)/the number of domain of the neighbors of gene_A )+((the neighbors of gene_A and the neighbors of potential gene)/the number of domain of the neighbors of potential gene)). In the later version, a p-value will be calculated to evaluate the significance of the best potential ortholog. Currently, this module can be used only for two plant genomes, Arabidopsis and Poplar.

rank_pgene

  $results = $obj->rank_pgene($wgene, $pgenes)
Parameter and return types
$wgene is a wgene
$pgenes is a pgenes
$results is a pgene2score
wgene is a string
pgenes is a reference to a list where each element is a pgene
pgene is a string
pgene2score is a reference to a hash where the key is a pgene and the value is a score
score is a float
Description

'rank_pgene' receives two parameters:wgene and pgenes. wgene is a well-annotated gene in organism A. pgenes is a list of potential functional orthologs in organism B. This function first retrieves the co-expression neighbors through KBase Networks API. Then compare the similarity of those neighbors between potential ortholog and the well-annotated gene. PFAM domain information is utilized to do the comparsion. Basically, this function count the shared PFAM domains between two groups of neighbors. The more the shared domains, the higher the score is. When we are using the KBase networks API to get the co-expression neighbors for each gene, we have to select the dataset. Currently, the dataset is selected by default. Later, we can give users the right to select their interested dataset.

get_pfam

  $results = $obj->get_pfam($ids)
Parameter and return types
$ids is an ids
$results is an ids2domains
ids is a reference to a list where each element is an id
id is a string
ids2domains is a reference to a hash where the key is an ids and the value is a domains
domains is a reference to a list where each element is a domain
domain is a string
Description

'get_pfam' receives kbase locus id and returns its PFAM domain list. Current only Arabidopsis and Poplar is supported. The PFAM annotation is download from biomart and stored in devdb1.newyork.kbase.us:kbase_plant:pfam_annotation. This function is only used as a support for the rank_pgene.

Introduction

The KBase Central Store (CS) contains networks data pulled from a variety of public databases. This tutorial will teach you:

  • How to obtain the general information on what networks types, datasets, and dataset sources are currently available in KBase CS.
  • How to access the KBase CS to extract specific subsets of network data using the KBase Networks API 1.0

Learning what network data are out there

Classes of network nodes and edges

KBase uses a very general definition of a network. We currently allow three classes of network nodes: gene, protein, and cluster or a group of gene/proteins such as e.g., a multi-protein complex or a metabolic pathway. Network edges correspondingly come in five main classes: gene-gene, protein-protein, gene-cluster, protein-cluster, and cluster-cluster. In the future, we plan to expand the number of classes of networks nodes and edges in KBase by adding e.g., genome-genome nodes for ecosystems deduced from metagenomic data.

Functional types of networks in KBase

KBase network data currently come in four distinct types:
  • REGULATORY_NETWORK - a gene-gene edge represents a transcriptional regulatory interaction between two genes directed from a transcriptional regulator to its target. This edge can be positive or negative, representing activation and repression correspondingly. An alternative representation of a regulatory network uses gene-cluster edges representing a gene’s membership in a regulon (cluster).
  • PROT_PROT_INTERACTION - an undirected protein-protein edge in this network represents a physical interaction (specific binding) between a pair of proteins. Alternatively, protein-cluster edges represent membership of a protein in a multi-protein complex (cluster).
  • METABOLIC_SUBSYSTEM - a directed gene-gene edge in this network indicates that two genes encoding enzymes are connected to each other via a metabolite: the direction of this edge is from the enzyme having this metabolite as a product to the one having it as one of its substrates. Alternatively, edges of the type gene-cluster represent membership of a gene in a given metabolic subsystem (cluster) such as e.g. KEGG or a BioCyc metabolic pathway.
  • FUNCTIONAL_ASSOCIATION - an edge in this type of network indicates that two genes are functionally associated. Functional association is deduced by integrating various data types including but not limited to co-expression, co-occurrence and co-localization in sequenced genomes, and co-occurence in abstracts of published papers, etc. Among examples of this type of networks are AraNet (http://www.functionalnet.org/aranet/) for Arabidopsis and StringDB (http://string-db.org) for most sequenced microbial and plant genomes.
To find the current list of network types in KBase you can use the command allNetworkTypes

Dataset sources for networks in KBase CS

Network data in KBase comes from a variety of public sources. The current list includes:
  • REGPRECISE - Curated database of transcriptional regulons reconstructed by the comparative genomic approach in a wide variety of prokaryotic genomes (http://regprecise.lbl.gov)
  • REGTRANSBASE - Curated database of regulatory interactions in prokaryotes capturing the knowledge in published scientific literature (http://regtransbase.lbl.gov)
  • CMONKEY - Integrated biclustering of heterogeneous genome-wide datasets for the inference of global regulatory networks (http://baliga.systemsbiology.net/drupal/content/cmonkey)
  • MAK_BI_CLUSTER - Massive associative k-biclustering
  • INTACT - Plants Protein-Protein Interactions (http://www.ebi.ac.uk/intact)
  • AGRIS - Database of transcriptional regulation in Arabidopsis collected by the Grotewold lab at Ohio State University (http://Arabidopsis.med.ohio-state.edu). Their AtRegNet dataset currently contains 11,355 direct interactions between TFs and target genes.
  • PLANTNET - Probabilistic functional association gene network in several plant species (currently Arabidopsis and Poplar) constructed by a modified Bayesian integration of 24 types of "omics" data from multiple organisms, with each data type weighted according to how well it links genes that are known to function together. It is based on AraNet (http://www.functionalnet.org/aranet) and RiceNet (http://www.functionalnet.org/ricenet) developed by collaboration among the Lee Lab at Yonsei University, Korea; the Marcotte Lab at University of Texas at Austin, USA; the Rhee Lab at Carnegie Institution for Science, Stanford, USA; and the Ronald Lab at UC Davis.
  • MODELSEED - Metabolic subsystems and models (http://www. theseed.org/models).
  • MO - A resource for browsing and comparing microbial genomes (http://www.microbesonline.org).
  • ECOCYC - EcoCyc is a scientific database for the bacterium Escherichia coli K-12 MG1655 (http://www.ecocyc.org).
  • PPI - Other Protein-Protein Interaction datasets not yet included in IntAct.
To find the current list of dataset sources in the KBase use the command allDatasetSources.

Networks datasets in KBase CS

Each dataset source listed by allDatasetSources may contribute more than one network dataset to the KBase. Examples include organism-specific networks extracted from the IntAct, different plant species in PLANTNET, and different versions of bi-clusters. To find the current list of datasets in the KBase use the command allDatasets.

Learning how to extract specific subsets of network data

Finding first neighbors of a node

The API command buildFirstNeighborNetwork generates the network connecting user-specified node to its first neighbors in user-specified list of network datasets and network data types. To use it efficiently you need to be familiar with how to find the genes and genomes you are interested in described in Getting What You Need from the CS Using Command-Line Scripts tutorial.

For example to find KBase ID for Escherichia coli K12 genome we used the command:

      all_entities_Genome -f scientific_name | grep Escherichia

and among all E. coli strains selected the KBase ID for Escherichia coli K12. Appropriately it is  'kb|g.0'.

To find the list of genes in this genome we used:

        echo 'kb|g.0' | genomes_to_fids | grep 'peg' | fids_to_functions

To find the gene encoding RecA protein we added grep 'RecA' to the previous comand:
    
     echo 'kb|g.0' | genomes_to_fids | grep 'peg' | fids_to_functions | grep 'RecA'

The KBase ID of the RecA gene is ‘kb|g.0.peg.1356’

Another example is a bacterium "Shewanella oneidensis MR-1" with KBase ID: “kb|g.20848”. To discover what network datasets include genes from this genome we run:

   allDatasets | grep "kb|g.20848".

One of the datasets listed in the output of this command has KBase ID kb|netdataset.kb|g.20848.regulome.0
It is a regulatory network from the REGPRECISE database. We are interested in a gene encoding isocitrate lyase.
To find its KBase ID we run
         echo 'kb|g.20848' | genomes_to_fids | fids_to_functions | grep "isocitrate lyase"

which shows that in "Shewanella oneidensis MR-1" this gene’s KBase ID is "kb|g.20848.CDS.1671".

The first neighbor network is found by running:
          echo "kb|g.20848.CDS.1671" | buildFirstNeighborNetwork "kb|netdataset.kb|g.20848.regulome.0" GENE_CLUSTER

A variant of this command buildFirstNeighborNetworkLimtedByStrength uses and additional cutOff parameter setting a lover threshold on the strength of edges to be considered. This cutOff is a part of some but not all of network datasets such as e.g. functional association networks in PlantNet, or co-expression networks.The format to use this command is:
     echo KBase_gene_ID1| buildFirstNeighborNetworkLimtedByStrength KBase_network_dataset_ID1 GENE_GENE cutOff;

Finding a subnetwork connecting user-specified set of nodes

The command buildInternalNetwork returns the subnetwork connecting user-specified list of nodes and constructed from edges in the user-specified list of network datasets. The format to use this command for 3 nodes and one network dataset is:

     echo KBase_gene_ID1 KBase_gene_ID2 KBase_gene_ID3 buildInternalNetwork KBase_network_dataset_ID1 GENE_CLUSTER;

The command buildInternalNetworkLimitedByStrength adds a numerical cutOff parameter setting the lower bound on the strength of edges to be considered.
It can be invked by using:

    echo KBase_gene_ID1 KBase_gene_ID2 KBase_gene_ID3 buildInternalNetworkLimitedByStrength KBase_network_dataset_ID1 GENE_CLUSTER cutOff;

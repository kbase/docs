Introduction

The KBase IRIS interface allows one to run all of the KBase command line scripts and some useful UNIX tools from a web-based interface without downloading any software. The interface keeps track of who you are and stores your results and history on our KBase IRIS server. The prototype version of the IRIS interface is located here.

Using the IRIS Interface

The initial page looks like this:
   

Your first action should be to log in. Click "Sign In," which is located in the upper right corner of the interface. In the popup box that appears, log in using the KBase username and password selected when you signed up for your user account. After successfully logging in, you should see the text:

Authenticated as username

When you are ready to log out at the end of your session, simply type logout in the command entry box.

Along the left side, you will notice a list of available script and command categories that you can use. Click on a command category to expand it and view the scripts and commands. Click on a command to easily place it into the command entry box.

To learn more about each, click on the ? button next to the command name. Alternatively, enter the script or command followed by a -h. For example, if you enter the Workspace script kbws-addtype -h, you should see the output:

kbws-addtype <Object type> [-?eh] [long options...]
 -e --showerror   Set as 1 to show any errors in execution
 -? -h --help     Print this usage information

The commands and scripts availalble on the left side of the interface contain the commands that are used most frequently, but this is not an exhaustive set of all commands available. To view all available commands, type commands in the command entry box.

When entering commands, a few shortcuts are available. Use the up and down arrows to scroll through your command history. Use the right arrow key to autocomplete a partially typed command.

UNIX Commands Available in IRIS

In IRIS, you can also use the following commands which are simply run in a UNIX shell on the server and use the full semantics of the UNIX commands:
                   
        sort , grep , cut , cat, head, tail, date, echo, wc, tr

File system commands are also available, but are re-implemented on the server as very limited version of the originals.

These are:

    mkdir DIR
    rmdir DIR (Removes non-empty directories without asking)
    rm FILE
    mv FROM TO
    cp FROM TO
    cd DIR
    ls [DIR]

Additional Features of IRIS

File Browser

IRIS maintains a file system for your use. You can upload your own data files (using the up-arrow button in th lower right corner of the File Browser box), or create your own via output redirection using the > redirection symbol. Let's say you wanted to create a file of Streptococcus genomes; you would type

    all_entities_Genome -f scientific_name | grep "Streptococcus pneumonia"  > all_strep

This creates a file with the name all_strep in your IRIS user space. You should see this filename appear in your File Browser section of the interface. You can also see your stored files by using the ls command. Stored files persist from session to session.


You can view individual files by selecting them and then clicking on the "view selected file icon," represented by the magnifying glass. The file will open in a new window.

The File Browser also provides shortcut icons for creating directories and deleting files. To move files, use the common UNIX commands from the previous section.

Running Processes

IRIS runs commands asynchronously, which means that several commands can be run simultaneously. You need not wait for one command to complete prior to invoking another.

Running commands are listed in the Running Processes section of the interface. Locate the section:

While observing the contents of this section, type the command all_entities_Genome. All commands that are invoked and that are still running should be listed in this box, and then disappear upon completion.

Understanding KBase Commands and Output

Most of the KBase command-line tools take in a file containing a tab-separated table and output a modified table. The most common modification is the addition of one of more columns. We create "pipelines" of these tools to implement fairly complex transformations leading to the final table containing the desired output. For example, consider the following little pipeline:

           all_entities_Genome -f scientific_name | grep "Streptococcus pneumoniae"

The all_entities_Genome command is thought of as producing a table in which the first column (by default) is the genome ID, and any extra columns come from the arguments of the command. In this case, we get a 2-column table:

           [genome-ID,genome-name]

This 2-column table is fed into a UNIX command called grep, which keeps lines that match its argument. In this case, the grep extracts rows in the table that match the string "Streptococcus pneumoniae." Thus, we get as a result a 2-column table in which each row contains "Streptococcus pneumoniae." 

      
   kb|g.3478 Streptococcus pneumoniae SPN034183
   kb|g.3264 Streptococcus pneumoniae GA47901
   kb|g.162 Streptococcus pneumoniae R6
   kb|g.3483 Streptococcus pneumoniae SPN032672
   ...
   kb|g.3482 Streptococcus pneumoniae SPN1041
   kb|g.1577 Streptococcus pneumoniae CDC3059-06
   kb|g.3259 Streptococcus pneumoniae GA04375
